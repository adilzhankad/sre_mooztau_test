# MoozTau — Дизайн бриф для Pencil / Tldraw / Figma

## О продукте

MoozTau — мобильное приложение (iOS/Android + веб) для управления производством и продажами холодильного оборудования в Казахстане. Используется ежедневно менеджерами, дилерами и работниками цеха — в основном с телефона.

## Целевая аудитория

- Менеджеры по продажам (25-45 лет, Казахстан) — создают заказы с телефона, часто в дороге
- Руководители дилеров — смотрят аналитику, управляют менеджерами
- Работники цеха — обновляют статусы производства, часто грязные руки → крупные кнопки
- Директор — общая аналитика, контроль

## Брендбук (с сайта mooztau.kz)

Дизайн приложения должен быть визуально связан с основным сайтом https://mooztau.kz/

**Цвета бренда:**
- Основной синий: `#3e6cb6` (используется на сайте для кнопок, акцентов)
- Тёмный фон: `#14181e` (dark mode, хедер, футер)
- Светлый фон: `#f6f7f8` (фон контента)
- Белый: `#ffffff` (карточки)
- Зелёный: `#22c55e` (успех, приход, подтверждение)
- Красный: `#ef4444` (ошибки, расход, отказ)
- Оранжевый: `#f59e0b` (предупреждения, в работе)

**Шрифт:** Inter, sans-serif (тот же что на сайте)

**Общий стиль:**
- Корпоративный/промышленный минимализм — как на mooztau.kz
- Скругления: 1rem (16px) для карточек, 1.5rem (24px) для крупных блоков
- Иконки: Material Symbols Outlined (как на сайте) или Heroicons
- Поддержка тёмной темы (сайт поддерживает dark mode)

## Стиль приложения

- **Mobile-first** — основное использование с телефона
- Визуально единый с mooztau.kz — тот же синий `#3e6cb6`, тот же Inter, тот же стиль карточек
- Крупные touch-friendly элементы (минимум 44px тач-зона)
- Русский и казахский язык
- Логотип MoozTau — SVG с сайта (такой же как в хедере mooztau.kz)
- Кнопки: скругление 1rem, основной цвет `#3e6cb6`, hover потемнее
- Без лишних украшений — утилитарный дизайн для работы

## Экраны для дизайна

### 1. Логин
- Поле телефон (маска +7 XXX XXX XX XX)
- Поле пароль
- Кнопка "Войти"
- Логотип MoozTau сверху

### 2. Заказы (главный экран)
- Поиск сверху
- Фильтры: статус (теги-кнопки горизонтально), период
- Список заказов — карточки:
  - Номер заказа (MZ-000042)
  - Имя клиента, регион
  - Сумма (крупно)
  - Статус (цветной бейдж)
  - Дата
- FAB кнопка "+" для создания заказа
- Pull-to-refresh

### 3. Создание заказа
- Шаги/секции:
  1. **Клиент**: Имя, Телефон, Регион (выпадающий), Адрес
  2. **Товары**: Добавить позицию → модель (выбор из каталога), количество, размеры, цвет, цена
  3. **Итого**: Сумма, дедлайн, договор
- Каждая позиция товара — карточка с возможностью удалить
- Кнопка "Создать заказ" внизу (sticky)

### 4. Детали заказа
- Шапка: номер, статус (крупный цветной бейдж), сумма
- Секция "Клиент": имя, телефон (кликабельный → звонок), адрес
- Секция "Товары": список позиций
- Секция "Оплаты":
  - Прогресс-бар (оплачено / всего)
  - Список платежей
  - Кнопка "Добавить оплату"
- Секция "История": таймлайн изменений статусов
- Кнопка смены статуса (если роль позволяет)

### 5. Каталог продукции
- Категории горизонтально (табы): Выносной, Встроенный, Морозильник, Агрегат, Дверь
- Сетка карточек моделей:
  - Название модели (J-2000)
  - Цена (РРЦ или дилерская — зависит от роли)
  - Единица (шт/метр)
- Клик на карточку → детали модели

### 6. Цех (Фабрика)
- Дашборд сверху: 4 карточки с числами
  - В очереди / В работе / Готово / Отгружено
- Список заказов в работе:
  - Номер, модель, размеры, цвет, дедлайн
  - **Нет цен** — цех не видит деньги
  - Кнопка смены статуса (свайп или кнопка)

### 7. Склад
- Список: модель, цвет, количество на складе
- Поиск по модели
- Кнопка "+" для добавления
- Количество — крупный цифры

### 8. Аналитика (SUPER_ADMIN)
- Карточки сверху: Заказов, Выручка, Средний чек, Клиентов
- Графики:
  - Выручка по месяцам (столбцы)
  - Заказы по статусам (пирог)
  - Топ дилеров (горизонтальные бары)
  - Топ моделей (горизонтальные бары)
- Каждый блок — отдельная секция с заголовком

### 9. Пользователи
- Список пользователей с аватарками (инициалы)
- Роль (цветной бейдж)
- Организация
- Активен/неактивен (тогл)
- Кнопка "+" для добавления

### 10. Организации
- Список каналов продаж
- Тип (HQ/Филиал/Дилер) — цветной бейдж
- Регион
- Контакты

### 11. Профиль
- Аватар (инициалы), Имя, Роль, Организация
- Кнопка "Выйти"

## Навигация

Нижний таб-бар (5 табов, зависит от роли):

**SUPER_ADMIN:** Заказы | Каталог | Фабрика | Аналитика | Ещё (Пользователи, Организации, Профиль)

**DEALER_ADMIN:** Заказы | Каталог | Пользователи | Организация | Профиль

**DEALER_MANAGER:** Заказы | Каталог | Профиль

**FACTORY_ADMIN:** Фабрика | Склад | Профиль

## Типографика

- Заголовки: Inter Bold / SF Pro Bold
- Тело: Inter Regular / SF Pro Regular
- Числа (суммы): Monospace или tabular figures
- Размеры: H1 24px, H2 20px, Body 16px, Caption 13px

## Иконки

- Линейные (outline), 24px
- Стиль: Heroicons или Phosphor Icons
- Нижний таб-бар: filled для активного, outline для неактивного

## Компоненты

- **Карточка заказа**: скругление 12px, тень subtle, отступы 16px
- **Бейдж статуса**: скругление pill (999px), padding 4px 12px, bold text
- **Кнопка основная**: скругление 12px, height 48px, full-width
- **Инпуты**: скругление 10px, border 1px gray-200, height 44px
- **FAB**: 56px круг, тень, правый нижний угол
- **Bottom tab bar**: height 64px + safe area, 5 табов

## Ключевые UX принципы

1. **Одно действие на экран** — не перегружать
2. **Крупные тач-зоны** — работники цеха используют с грязными руками
3. **Быстрое создание заказа** — максимум 3 шага
4. **Статус виден сразу** — цветной бейдж на каждой карточке
5. **Сумма крупно** — деньги всегда на виду
6. **Оффлайн-готовность** — показывать кешированные данные
