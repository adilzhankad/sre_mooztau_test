# MoozTau — Техническое задание

## Что это

Платформа управления производством и продажами холодильного оборудования. Казахстан, ~30 сотрудников, 10 каналов продаж, 640+ заказов.

---

## Роли пользователей

**SUPER_ADMIN** — руководство MoozTau. Видит всё, управляет всем. Создаёт дилеров, устанавливает цены, смотрит аналитику по всей компании.

**DEALER_ADMIN** — руководитель дилера/филиала. Видит только свой канал. Добавляет своих менеджеров, устанавливает им лимиты, создаёт заказы.

**DEALER_MANAGER** — продавец у дилера. Видит только свои заказы. Создаёт заказы, не видит дилерскую цену (только рекомендуемую розничную).

**FACTORY_ADMIN** — начальник цеха. Видит заказы в производстве. Меняет статусы, управляет складом готовой продукции. Не видит цены.

**FACTORY_WORKER** — рабочий цеха. Видит свои задачи, обновляет статус выполнения.

---

## Организации (каналы продаж)

Три типа:
- **HQ** — MoozTau (головной офис, один)
- **BRANCH** — собственные филиалы (MT Астана, MT Алматы, Каспи Магазин, B2B)
- **DEALER** — внешние дилеры (Ayza, Umag Шым, Umag Тараз, Диллер Болат, Айзберг)

У каждого дилера есть **кредитный лимит** — если задолженность превышает лимит, новые заказы блокируются.

## Ценообразование

Два уровня цен:
- **Дилерская цена** — по сколько MoozTau продаёт дилеру (видят SUPER_ADMIN и DEALER_ADMIN)
- **РРЦ** (рекомендуемая розничная) — рекомендация для конечного покупателя (видят все кроме цеха)

Дилер может продавать по любой цене. При создании заказа он указывает свою фактическую цену продажи. Цех вообще не видит цены — только что произвести.

## Заказы — основной процесс

### Создание заказа

Кто создаёт: DEALER_MANAGER, DEALER_ADMIN, SUPER_ADMIN.

Обязательные поля:
- Имя клиента
- Телефон клиента
- Регион доставки
- Точный адрес
- Хотя бы одна позиция товара (модель, количество, размеры, цвет, цена)

Без полных данных клиента заказ не принимается.

При создании:
- Автоматический номер (MZ-000001, MZ-000002...)
- Автоматический расчёт суммы из позиций
- Проверка кредитного лимита дилера
- Запись в аудит-лог

### Жизненный цикл заказа

```
НОВЫЙ → ПОДТВЕРЖДЁН → В ПРОИЗВОДСТВЕ → ГОТОВ → ДОСТАВКА → ДОСТАВЛЕН → ПРИНЯТ → ЗАВЕРШЁН

В любой момент SUPER_ADMIN может: ОТКАЗ или ВОЗВРАТ
```

| Переход | Кто делает | Что происходит |
|---------|-----------|---------------|
| Новый → Подтверждён | SUPER_ADMIN | HQ подтвердил, заказ идёт в очередь |
| Подтверждён → В производстве | FACTORY_ADMIN | Цех взял в работу |
| В производстве → Готов | FACTORY_ADMIN/WORKER | Произвели, на складе |
| Готов → Доставка | SUPER_ADMIN | Отправили клиенту |
| Доставка → Доставлен | SUPER_ADMIN/DEALER_ADMIN | Товар прибыл |
| Доставлен → Принят | DEALER_ADMIN/MANAGER | Клиент подписал приёмку |
| Принят → Завершён | SUPER_ADMIN | Только после полной оплаты |

При статусе "Принят" — автоматически ставится дата приёмки и дата окончания гарантии (через 1 год).
При статусе "Завершён" — система проверяет что оплата 100%, иначе не даёт завершить.

### Заказ содержит

- Позиции товара (модель, размеры, цвет, количество, цена за единицу, итого)
- Платежи (сколько оплачено, когда, каким способом)
- Историю изменений (кто что когда менял)
- Привязку к организации и менеджеру

---

## Оплаты

К каждому заказу можно добавлять оплаты:
- Сумма
- Дата
- Способ (Kaspi, Halyk, БЦК, Bereke, Forte, наличные)

Система считает: сколько оплачено, сколько осталось.

**Автоматическая связь с финансами:** когда оплата добавляется в заказ — автоматически создаётся запись "Приход" в финансовом модуле с привязкой к номеру заказа.

---

## Финансы (приход/расход)

Отдельный модуль для учёта всех денег компании.

**Транзакции** — каждая запись это либо приход, либо расход:
- Дата, сумма, валюта (KZT по умолчанию)
- Банковский счёт (Kaspi, Halyk, БЦК, Bereke, Forte, Наличными)
- Категория расходов (4 уровня вложенности)
- Инициатор (кто потратил/получил)
- Комментарий
- Привязка к заказу (если есть)

**Категории расходов** — дерево из 4 уровней:
```
Уровень 1: Подразделение     → Цех Кулан, Офис Алматы, Склад Кулан, Продажа, Финансы...
Уровень 2: Тип расхода       → Транспорт, Зарплата, Питания, Материал...
Уровень 3: Подкатегория      → ГСМ, Аванс, Обед, Профиль дверной...
Уровень 4: Детализация       → 783APE08, Сварщик, Директор...
```

**Банковские счета** — автоматический баланс. Приход увеличивает, расход уменьшает.

**Отчёты:**
- Сводка за период (приход/расход/баланс)
- Разбивка по категориям
- Разбивка по инициаторам
- Экспорт в Excel

## Цех (производство)

Цех видит:
- Очередь заказов (подтверждённые, ждут производства)
- Заказы в работе
- Готовую продукцию
- Дедлайны и просрочки

Цех НЕ видит: цены, финансы, данные других каналов.

**Дашборд цеха:**
- Сколько в очереди
- Сколько в производстве
- Сколько готово
- Сколько просрочено

---

## Склад (инвентарь)

Учёт готовой продукции по цехам (Кулан, Тараз):
- Модель, цвет, количество
- Статус: на складе / зарезервировано / отгружено

Если товар есть на складе — доставка быстрее (не нужно производить). Если нет — заказ идёт в производство, доставка дольше.

---

## Аналитика (только SUPER_ADMIN)

- **Общая сводка:** заказы, выручка, средний чек, рост по сравнению с прошлым периодом
- **Выручка:** по месяцам, по каналам, по менеджерам
- **Заказы:** по статусам, по регионам, по моделям
- **Дилеры:** сравнение каналов (заказы, выручка, долг, кредитный лимит)
- **География:** продажи по регионам Казахстана
- **Продукция:** популярность моделей
- **Платежи:** полученные, задолженность, по способам оплаты
